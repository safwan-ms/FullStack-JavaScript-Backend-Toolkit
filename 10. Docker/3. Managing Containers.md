# 3. Managing Containers

## Logs
```bash
docker logs my-nginx
```
View logs from a container.

```bash
docker logs -f my-nginx
```
Follow logs in real time (`-f` = follow).

---

## Stop & Start
```bash
docker stop my-nginx
```
Gracefully stop a running container.

```bash
docker start my-nginx
```
Start a stopped container.

```bash
docker restart my-nginx
```
Stop and start a container in one command.

---

## Remove
```bash
docker rm my-nginx
```
Remove a stopped container.

```bash
docker rm -f my-nginx
```
Force remove a running container without stopping it first.

---

## Execute a Command Inside a Running Container
```bash
docker exec -it my-nginx bash
```
Open an interactive shell inside a running container.

---

## Inspect & Stats
```bash
docker inspect my-nginx
```
Show detailed information (IP, mounts, config, etc.) about a container.

```bash
docker stats
```
Live resource usage stats (CPU, memory) for all running containers.

---

## Clean Up Everything
```bash
docker system prune
```
Remove all stopped containers, unused images, and unused networks.
