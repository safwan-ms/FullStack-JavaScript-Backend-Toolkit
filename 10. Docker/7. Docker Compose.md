# 7. Docker Compose

Docker Compose lets you define and run **multi-container** applications using a single `docker-compose.yml` file.

---

## Example — Node.js + MongoDB

```yaml
version: "3"

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - MONGO_URI=mongodb://mongo:27017/mydb
    depends_on:
      - mongo

  mongo:
    image: mongo
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db

volumes:
  mongo-data:
```

---

## Core Commands

```bash
docker compose up
```
Start all services defined in `docker-compose.yml`.

```bash
docker compose up -d
```
Start all services in detached (background) mode.

```bash
docker compose down
```
Stop and remove all containers, networks created by Compose.

```bash
docker compose down -v
```
Also remove volumes when bringing everything down.

```bash
docker compose logs -f
```
Follow logs from all services.

```bash
docker compose ps
```
List containers managed by Compose.

```bash
docker compose build
```
Build (or rebuild) images defined in the Compose file.

```bash
docker compose exec app bash
```
Open a shell inside a running Compose service.
