# 4. Volumes

Volumes are used to **persist data** outside the container lifecycle.

---

## Create & List Volumes
```bash
docker volume create my-volume
```
Create a named volume.

```bash
docker volume ls
```
List all volumes.

```bash
docker volume rm my-volume
```
Remove a volume.

---

## Mount a Volume into a Container
```bash
docker run -d --name my-nginx -v my-volume:/usr/share/nginx/html nginx
```

| Part | Description |
|------|-------------|
| `my-volume` | The named volume on the host |
| `/usr/share/nginx/html` | The path inside the container to mount it to |

---

## Bind Mount (Map a Local Folder)
```bash
docker run -d --name my-nginx -v /home/user/html:/usr/share/nginx/html nginx
```
Maps a folder on your machine directly into the container.  
Changes on the host are instantly reflected inside the container.

---

## Inspect a Volume
```bash
docker volume inspect my-volume
```
Shows where the volume is stored on the host and other details.
