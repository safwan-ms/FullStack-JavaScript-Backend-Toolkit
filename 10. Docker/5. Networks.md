# 5. Networks

Docker networks allow containers to **communicate with each other**.

---

## List & Create Networks
```bash
docker network ls
```
List all networks.

```bash
docker network create my-network
```
Create a custom bridge network.

```bash
docker network rm my-network
```
Remove a network.

---

## Connect a Container to a Network
```bash
docker run -d --name my-nginx --network my-network nginx
```
Run a container and attach it to a specific network.

```bash
docker network connect my-network my-nginx
```
Connect an already running container to a network.

```bash
docker network disconnect my-network my-nginx
```
Disconnect a container from a network.

---

## Inspect a Network
```bash
docker network inspect my-network
```
Shows which containers are connected and their IPs.

---

## Network Types

| Type | Description |
|------|-------------|
| `bridge` | Default. Containers on same bridge can talk to each other |
| `host` | Container shares the host's network directly |
| `none` | No networking |
