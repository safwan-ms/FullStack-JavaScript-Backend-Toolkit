# 6. Dockerfile

A **Dockerfile** is a script of instructions to build a custom Docker image.

---

## Example — Node.js App

```dockerfile
# Base image
FROM node:18

# Set working directory inside the container
WORKDIR /app

# Copy package files and install dependencies
COPY package*.json ./
RUN npm install

# Copy the rest of the source code
COPY . .

# Expose the port the app runs on
EXPOSE 3000

# Command to start the app
CMD ["node", "index.js"]
```

---

## Common Dockerfile Instructions

| Instruction | Description |
|-------------|-------------|
| `FROM` | Base image to build from |
| `WORKDIR` | Set the working directory inside the container |
| `COPY` | Copy files from host into the container |
| `RUN` | Run a command during the image build |
| `EXPOSE` | Document which port the container listens on |
| `ENV` | Set environment variables |
| `CMD` | Default command to run when the container starts |
| `ENTRYPOINT` | Like CMD but not overridable from `docker run` |

---

## .dockerignore

A `.dockerignore` file tells Docker which files to **exclude** when building an image — keeps the image small and avoids leaking secrets.

```
node_modules
.env
.git
*.log
dist
```

> Always add `.dockerignore` to every project that has a Dockerfile.

---

## Build & Run Your Image

```bash
docker build -t my-app .
```
Build an image from the Dockerfile in the current directory and tag it `my-app`.

```bash
docker run -d -p 3000:3000 --name my-app my-app
```
Run a container from your custom image.

```bash
docker build -t my-app:v2 .
```
Build with a version tag.
