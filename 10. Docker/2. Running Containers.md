# 2. Running Containers

## Basic Run
```bash
docker run nginx
```
Run a container from the nginx image (downloads it if not already pulled).

---

## Run in Background (Detached)
```bash
docker run -d nginx
```
Run the container in the background (`-d` = detached mode).

---

## Run with a Name
```bash
docker run -d --name my-nginx nginx
```
Give the container a custom name instead of a random one.

---

## Run with Port Mapping
```bash
docker run -d --name my-nginx -p 8000:80 nginx
```

| Flag | Description |
|------|-------------|
| `-d` | Detached mode — runs in the background |
| `--name my-nginx` | Give the container a custom name |
| `-p 8000:80` | Map host port `8000` → container port `80` |

> Visit `http://localhost:8000` to see nginx running.

---

## Run with Environment Variables
```bash
docker run -d --name my-app -e NODE_ENV=production my-image
```
Pass environment variables into the container with `-e`.

---

## Run Interactively
```bash
docker run -it ubuntu bash
```

| Flag | Description |
|------|-------------|
| `-i` | Interactive — keep STDIN open |
| `-t` | Allocate a terminal (TTY) |

---

## Run and Auto Remove on Exit
```bash
docker run --rm hello-world
```
Automatically removes the container when it stops.

---

## Run with an .env File
```bash
docker run -d --name my-app --env-file .env my-image
```
Load all environment variables from a `.env` file into the container.

```
# .env
NODE_ENV=production
PORT=3000
DB_URL=mongodb://localhost:27017/mydb
```

> Use `--env-file` instead of multiple `-e` flags when you have many variables.
